# 🛠️ 에이전트 스킬 및 전역 지식 (SKILL.md)

이 문서는 프로젝트 과정에서 도입되어 성공적으로 검증된 핵심 기술과 패턴을 영구 기록하여, 향후 에이전트가 다른 작업에 재사용할 수 있도록 돕습니다.

## 1. 저사양 CPU 극한 최적화 (Context & Text Limit Tuning)
- **개요**: N150 등 저전력 100% CPU 구동 환경에서 LLM 추론 속도를 극대화하기 위한 파라미터 튜닝 기법입니다.
- **적용 스킬**:
  - `num_ctx` 축소: 기존 2048에서 **1024**로 절반 수준으로 축소하여 메모리 스와핑 방지 및 연산량 감소.
  - 전처리 텍스트 크기 축소: 메일 본문의 길이를 1200자에서 **600자**(`substring(0, 600)`)로 대폭 제한하여 컨텍스트 윈도우 한계 내에서 빠르고 간결한 처리를 강제.

## 2. 공격적인 광고 메일 클린업 패턴 (30-day Sweeping)
- **개요**: 불필요한 메일 누적을 방지하고 요약 엔진의 부하를 줄이기 위한 시프트-레프트(Shift-Left) 최적화입니다.
- **적용 스킬**:
  - `newer_than:30d subject:(광고)` 와 같은 강력한 30일치 범위 쿼리를 활용하여 범용 스팸 및 프로모션 메일을 요약 파이프라인 진입 전에 일괄적으로 휴지통으로 이동시킵니다.
  - 이를 통해 LLM API 호출 횟수를 대폭 감소시켜 비용 및 전체 실행 시간을 절약합니다.

## 3. 동적 모델 로깅 (Dynamic Model Logging)
- **개요**: 여러 로컬 LLM을 번갈아 사용할 때 어떤 모델이 동작 중인지 명확히 파악할 수 있도록 돕는 로깅 기법입니다.
- **적용 스킬**:
  - `process.env.OLLAMA_MODEL` 변수를 직접 읽어와(`const usedModel = process.env.OLLAMA_MODEL || 'llama3.2:1b';`) 텔레그램 알림 메시지와 콘솔 결과창에 주입합니다.
  - 이를 통해 런타임에 동적으로 변경되는 시스템 환경을 추적하기 용이하게 합니다.
